# Гайд користувача

## Відкриття sTask

Відкрийте sTask з менеджера Evolution CMS через зареєстрований модуль пакета.
Нова панель використовує локальні EvoUI assets. Якщо сторінка відкрилась без
стилів або скриптів, спочатку перевірте публікацію `stask` і `evo-ui` assets.

## Панель

Панель показує:

- задачі, що очікують;
- задачі в процесі;
- завершені задачі;
- помилкові задачі;
- загальну кількість задач;
- активних воркерів.

В таблиці останніх задач можна натиснути іконку ока або двічі клікнути рядок,
щоб відкрити деталі. Якщо помилок немає, блок останніх помилок не показується.

## Завдання

Вкладка Завдання має table і list режими. Таблиця потрібна для повного набору
колонок, список - для швидкого перегляду.

Фільтри:

- воркер;
- дія;
- статус;
- пріоритет;
- спроби;
- період створення.

Виводяться ID, назва воркера, ідентифікатор воркера, дія, статус, пріоритет,
прогрес, спроби, користувач запуску, повідомлення, створено, початок виконання,
завершено і оновлено.

Деталі відкриваються іконкою ока або double-click по рядку. У модалці видно
ідентифікацію задачі, воркер, статус, пріоритет, спроби, часові поля, лог,
метадані і результат.

## Воркери

Вкладка Воркери показує зареєстровані воркери: назву, опис, ідентифікатор,
область, активність, доступність класу, видимість, кількість задач, останній
статус, позицію і час оновлення.

Дії:

- редагувати налаштування;
- запустити стандартне завдання;
- активувати або деактивувати воркер.

Ті самі дії дублюються у верхньому toolbar для вибраного рядка. Для запуску або
деактивації спочатку виберіть воркер.

## Налаштування воркера

Налаштування містять клас, опис, активність, розклад і специфічні опції
воркера. Автоматичні воркери мають `taskMake` і можуть підтримуватися фоновою
командою.

## Логи

Логи є audit-панеллю задач. Вони використовують ту саму модалку деталей, що й
Завдання. Фільтруйте логи по воркеру, статусу і даті створення.

## Вбудовані воркери

sTask містить:

- стандартний task runner;
- Composer update worker;
- Artisan command worker.

Ці legacy widgets поступово переносяться на спільний task-runner contract, але
під час міграції запуск задач, прогрес, логи і результати мають працювати.

## Безпечна робота

- Після оновлення пакета перевидавайте assets.
- Після нових permissions перелогіньтесь у менеджері.
- Запуск воркерів, Composer update, Artisan commands і deactivate є діями, що
  змінюють стан системи.
